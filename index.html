Comandos Git: fetch, pull, merge, rebase, clone, push

Resumo rápido

  • git fetch: baixa dados do remoto e atualiza referências remotas sem mexer no HEAD nem no working tree.
  • git pull: fetch + merge (ou rebase) automático no branch atual.
  • git merge: une duas linhas de história criando um commit de merge.
  • git rebase: reescreve o histórico do branch atual para parecer que foi criado a partir da base mais recente (histórico linear).
  • git clone: cria um repositório local completo a partir de um remoto.
  • git push: envia commits locais para o remoto.

fetch vs pull

O que acontece?

// git fetch  
# Busca mudanças do remoto e atualiza origin/, sem mudar seu HEAD  
git fetch origin  

// git pull (padrão)  
# Faz fetch + merge no branch atual  
git pull origin main

Diagrama ASCII

Antes:  
local:  A -- B -- C (HEAD)  
remote: R -- S -- T (origin/main)  

git fetch origin  
Depois:  
local:  A -- B -- C (HEAD)  
remote: R -- S -- T (origin/main)  
# origin/main apontando para T; seu HEAD/branch não mudou

Merge vs Rebase

Merge

main:   A -- B -- C  
                 \
feature:          D -- E  
Merge result:     A -- B -- C -- M  
                          / \
                        D     E

Rebase

main:   A -- B -- C  
feature:        D -- E  
Rebase onto main: A -- B -- C -- D' -- E'

clone vs fetch vs pull

// Clone  
git clone   

# Em um repositório existente:
